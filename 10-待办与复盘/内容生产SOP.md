# 内容生产 SOP

## 先回答你的编辑问题

- 这些文件权限是可写的，当前权限是 `-rw-r--r--`
- 文件属主也是你自己：`wujames`
- 所以不是系统只读，也不是文件权限锁住了

如果你在 Obsidian 右侧不能改，通常是这几个原因：

- 你打开的是 `阅读视图`，不是 `编辑视图`
- 你开的是 `右侧预览面板`
- 这个标签页被 `Pin` 了，但本身仍在预览模式
- 你装了某些插件，把单击默认改成了预览

### Obsidian 里怎么切回可编辑

1. 打开任意文档
2. 看右上角有没有 `铅笔 / 书本` 图标
3. 点一下切到 `编辑模式`
4. 如果是右侧分屏，先点进那个面板，再切编辑模式
5. 如果还是不行，用命令面板搜：`Toggle edit/preview mode`

如果你愿意，我后面可以再给你补一份 `Obsidian 使用说明.md`。

## 这份 SOP 的目标

- 这不是原则清单，而是一套给 agent 或人直接执行的流程
- 每一步都必须有：`输入`、`动作`、`输出`、`落库位置`
- 每一步都可以拆成独立 skill

## 总流程图

```text
输入热点/想法
  -> S1 选题收集
  -> S2 选题筛选
  -> S3 角度定位
  -> S4 标题生成
  -> S5 口播成稿
  -> S6 配图分镜
  -> S7 发布组件打包
  -> S8 发布后复盘
  -> S9 入库与归档
```

## 执行约束

- 同一主题只保留一个正式文档
- 文档修改直接覆盖，版本交给 git
- 活跃稿放 `03-口播稿/`
- 旧稿放 `03-口播稿/归档/`
- 选题池放 `02-选题库/`
- 复盘放 `10-待办与复盘/`

## S1 选题收集

### 目标

- 把零散热点或想法，转成待判断的候选选题

### 输入

- 新闻热点
- 平台新功能
- 对标账号爆款角度
- 用户评论问题
- 你的即时判断

### 动作

1. 用一句话写下事件本身
2. 用一句话写下“老板为什么要关心”
3. 用一句话写下“最可能的结果感”
4. 临时记到素材区

### 输出格式

```text
事件：微信正式上线 ClawBot
老板相关：小老板第一次低门槛拥有数字员工
结果感：少招人 / 先让 AI 顶一部分活
```

### 落库位置

- `09-素材摘录/`

### 可拆成 skill

- `skill.topic-capture`

## S2 选题筛选

### 目标

- 判断这个话题是否值得进入生产流程

### 输入

- S1 输出的候选选题

### 动作

逐项打分，每项 0 或 1：

1. 是否和小企业老板直接相关
2. 是否能讲出明确结果感
3. 是否能在 3 秒内讲清冲突
4. 是否能自然接下一条内容
5. 是否和账号长期定位一致

### 规则

- 总分 `>=4`：进入生产
- 总分 `=3`：留在选题池观察
- 总分 `<=2`：丢弃

### 输出格式

```text
选题：微信接入龙虾，小老板真要少招人了
评分：5/5
结论：立即生产
```

### 落库位置

- 候选通过后写入 `02-选题库/`

### 可拆成 skill

- `skill.topic-score`

## S3 角度定位

### 目标

- 决定这一条到底从哪个角度打，不允许一条视频讲三个方向

### 输入

- 已通过筛选的选题

### 可选角度

- 热点翻译型
- 小老板结果型
- 危机提醒型
- 机会判断型
- 场景动作型

### 动作

1. 只选一个主角度
2. 写一句核心判断
3. 写一句不能讲什么

### 输出格式

```text
主角度：小老板结果型
核心判断：微信接入龙虾，不是功能升级，而是小老板第一次低门槛拥有数字员工
不讲：流程重构、大公司架构、技术细节
```

### 可拆成 skill

- `skill.angle-select`

## S4 标题生成

### 目标

- 生成 5 到 10 个标题，选 1 个主标题，2 个备选

### 输入

- S3 的主角度和核心判断

### 动作

1. 先写 1 个主标题
2. 再写 5 个备选
3. 过滤掉大词、黑话、长句

### 标题检查规则

- 不超过 20 个字优先
- 必须带老板视角或结果感
- 不出现“重构流程”“范式升级”这类生硬表达

### 输出格式

```text
主标题：微信都能养龙虾了，小老板真要少招人了
备选1：以后很多小老板，可能先不招人了
备选2：腾讯把龙虾接进微信，小老板的机会来了
```

### 落库位置

- 当前稿件文件内

### 可拆成 skill

- `skill.title-generate`

## S5 口播成稿

### 目标

- 生成可直接录制的口播稿

### 输入

- 标题
- 核心判断
- 角度类型

### 固定结构

1. `开头`：先讲新闻或现象
2. `翻译`：这对老板意味着什么
3. `解释`：为什么不是热闹，而是变化
4. `结果`：讲 2 到 3 个现实结果
5. `结尾`：一句强判断收尾

### 动作

1. 先写 60 秒版
2. 再压成 20 秒版
3. 再做一键复制版

### 口播检查规则

- 一句话只讲一个意思
- 优先短句
- 第一段不能先讲技术定义
- 全文必须回到老板结果

### 输出格式

```text
标题
封面文案
口播稿
20秒版
```

### 落库位置

- `03-口播稿/`

### 命名规则

- 直接可发：`热点可发布版-主题.md`
- 连发稿：`热点连发稿-主题.md`
- 普通稿：`热点口播稿-主题.md`

### 可拆成 skill

- `skill.script-build`

## S6 配图分镜

### 目标

- 给口播稿配上平台可分发的画面结构

### 输入

- 已完成的口播稿

### 动作

1. 给全文切成 5 到 7 段
2. 每段指定一种画面
3. 每段配 1 句屏幕短字幕

### 画面类型库

- 真人出镜
- 新闻截图
- 官方页面截图
- 聊天界面示意图
- 老板办公场景图
- 文件/表格/B-roll

### 输出格式

```text
0-3秒：真人出镜 + 大字标题
4-8秒：新闻截图 + 事件字幕
9-15秒：微信界面示意图 + 功能字幕
```

### 检查规则

- 每 3 到 5 秒必须切一次画面
- 不能全程纯口播黑底字
- 必须同时出现“新闻截图 + 产品界面 + 现实场景”中的至少两类

### 可拆成 skill

- `skill.visual-plan`

## S7 发布组件打包

### 目标

- 把一条视频发布所需的所有文本组件一次性准备完

### 输入

- 标题
- 口播稿
- 配图分镜

### 动作

输出以下组件：

1. 标题
2. 封面文案
3. 一键复制口播稿
4. 文案区首句
5. 评论区引导语

### 输出格式

```text
标题：...
封面：...
口播：...
文案区首句：...
评论区引导：...
```

### 可拆成 skill

- `skill.publish-pack`

## S8 发布后复盘

### 目标

- 把单条内容的反馈转成下一条内容的输入

### 输入

- 发布后的数据和评论

### 动作

记录以下 6 项：

1. 标题是否有效
2. 前 3 秒是否有效
3. 哪一句评论最多
4. 哪个画面最有信息量
5. 用户更关心机会、风险还是具体做法
6. 下一条最自然该接什么

### 输出格式

```text
标题表现：强/中/弱
高反馈句子：...
用户最关心：少招人
下一条建议：小老板最先让AI做哪3件事
```

### 落库位置

- `10-待办与复盘/`

### 可拆成 skill

- `skill.post-review`

## S9 入库与归档

### 目标

- 确保知识库不堆中间稿

### 动作

1. 如果是同主题迭代，覆盖原文件
2. 如果是新主题，按命名规则新建
3. 如果是旧稿不用了，移到 `03-口播稿/归档/`
4. 更新 `03-口播稿/目录导航.md`
5. 如有结构变化，更新 `知识库导航.md`

### 可拆成 skill

- `skill.asset-maintain`

## 最小执行链路

### 当天有热点时

```text
S1 选题收集
-> S2 选题筛选
-> S3 角度定位
-> S4 标题生成
-> S5 口播成稿
-> S6 配图分镜
-> S7 发布组件打包
```

### 发完后

```text
S8 发布后复盘
-> S9 入库与归档
```

## 当前建议执行顺序

1. 先执行 `S1-S7` 完成一条可发布内容
2. 直接优先使用 `03-口播稿/热点可发布版-微信接入龙虾小老板真要少招人了.md`
3. 发完立即执行 `S8`
4. 下一条内容从复盘结论继续推进
